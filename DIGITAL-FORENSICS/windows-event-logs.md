# Windows Event Logs

This file contains notes on Windows Event Logs and their importance during investigations.

---

## What are Windows Event Logs?

Windows Event Logs record activity occurring on a system.

These logs are valuable sources of evidence during security investigations and incident response activities.

---

## Common Log Categories

### Application Logs

Contain events generated by software applications.

Examples:
- Application crashes
- Software errors

---

### Security Logs

Contain security-related events.

Examples:
- User logins
- Failed login attempts
- Account changes

---

### System Logs

Contain events generated by the operating system.

Examples:
- System startup
- Shutdown events
- Driver issues

---

## Why Event Logs Matter

Event logs can help investigators:

- Determine what happened
- Identify suspicious activity
- Build timelines
- Support incident response efforts

---

## Investigation Examples

Examples of information that may be found:

- Successful logins
- Failed logins
- Account lockouts
- Service installations
- System reboots

---

## What I Learned

- Event logs provide important evidence during investigations.
- Security logs can reveal authentication activity.
- Event logs help establish timelines and support forensic analysis.
