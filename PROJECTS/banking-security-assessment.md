# Banking Security Assessment

## Overview

This project was completed as part of EPT232 – Ethical Hacking and Penetration Testing.

The objective was to analyse vulnerabilities identified during a penetration test of a simulated banking environment and develop security recommendations to reduce the risk of compromise.

---

## Key Findings

Several critical vulnerabilities were identified, including:

- A backdoored VSFTPD 2.3.4 service running on Port 21
- An unauthenticated bind shell on Port 1524
- Insufficient network segmentation
- Weak access control mechanisms

These weaknesses allowed an attacker to gain unauthorised access and escalate privileges within the environment.

---

## Security Recommendations

To address the identified risks, a Defence-in-Depth strategy was proposed, including:

- Next-Generation Firewalls (NGFW)
- Intrusion Prevention Systems (IPS)
- Multi-Factor Authentication (MFA)
- Role-Based Access Control (RBAC)
- Network Segmentation
- Zero Trust principles

## Key Lessons Learned

This project reinforced the importance of layered security controls and demonstrated how a single vulnerable service can lead to full system compromise if effective defensive measures are not implemented.

It also highlighted the role of network segmentation, access controls, and proactive monitoring in reducing the impact of security incidents.
